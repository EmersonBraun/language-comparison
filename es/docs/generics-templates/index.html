Genéricos y Plantillas

Los genéricos te permiten escribir código reutilizable y con tipos seguros. Así es como los diferentes lenguajes manejan la programación genérica.

      &self.value

}

          return self.value;
      }
  };
}

// Use void pointers for generic-like behavior

// Generic swap using void*
void swap(void* a, void* b, size_t size) {
  void* temp = malloc(size);
  memcpy(temp, a, size);
  memcpy(a, b, size);
  memcpy(b, temp, size);
  free(temp);
}

int a = 1, b = 2;
swap(&a, &b, sizeof(int));

<?php
// PHPStan template bounds

/**
* @template T of Comparable
* @param T $a
* @param T $b
* @return T
*/
function max_val($a, $b) {
  return $a->compareTo($b) > 0 ? $a : $b;
}

/**
* @template T of \Countable
* @param T $item
* @return int
*/
function getCount($item): int {
  return count($item);
}

  public function toString(): string;

// Trait bounds
fn max<T: PartialOrd>(a: T, b: T) -> T {
  if a > b { a } else { b }
}

// Multiple bounds
fn print_debug<T: std::fmt::Debug + Clone>(value: T) {
  println!("{:?}", value);
}

// Where clause (cleaner for complex bounds)
fn process<T>(value: T) -> String
where
  T: std::fmt::Display + Clone + Send,
{
  format!("{}", value)

// Associated type bounds
fn sum<I>(iter: I) -> i32
where
  I: Iterator<Item = i32>,

      return a

  return b

// comparable constraint
func Contains[T comparable](slice []T, item T) bool {

Puntos Clave​

  • Borrado de tipos vs monomorfización -- Java y TypeScript borran los genéricos en tiempo de ejecución (una representación compartida); C++, Rust, Swift y Go generan código especializado por tipo (monomorfización) para abstracción de costo cero.
  • Los sistemas de restricciones varían en expresividad -- Los traits de Rust y los concepts de C++ ofrecen límites ricos; los conjuntos de tipos de Go (~int | ~float64) y las restricciones de protocolo de Swift son más simples; el extends de Java y los comodines son más limitados.
  • Los lenguajes sin genéricos usan alternativas -- C usa void* y macros; JavaScript/Ruby dependen del duck typing; PHP usa anotaciones docblock para análisis estático; Zig usa parámetros de tipo en tiempo de compilación.
  • Los genéricos comptime (Zig) son únicos -- Zig pasa tipos como valores en tiempo de compilación, habilitando genéricos de costo cero sin una sintaxis separada de parámetros de tipo; los tipos son de primera clase en tiempo de compilación.
  • Python y PHP tienen genéricos tardíos -- Python 3.12+ y PHPStan/Psalm añaden sintaxis genérica; la verificación de tipos es solo estática—el comportamiento en tiempo de ejecución no cambia—a diferencia de los lenguajes compilados.
